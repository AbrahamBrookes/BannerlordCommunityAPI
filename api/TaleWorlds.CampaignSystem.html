Classes

CampaignBehaviorBase

The CampaignBehaviorBase class provides a basis for campaign behaviors. Campaign Behaviors describe the behavior of various entities in a given campaign mode. They are the basis of creating behaviors for occurrences that happen within a given game mode.

This class is extended into many, many classes that create the functionality of a campaign. All manner of different things that operate during gameplay are based on this class.

This class implements the ICampaignBehavior interface. Interfaces operate similarly to an abstract class in inheritance, but they are not a class per say, because they can not be initialized (alternatively called instanced) as a class can - that means you can't new an interface. An interface is also called a contract. This is a useful term because a real-life contract is used to define how an entity behaves.

Lets say you have a contract that says "every tuesday you will go to the pub". Then you will be "compatible" with anything that expects you to be at the pub on tuesdays. Maybe they have a happy hour at the pub on tuesday, but only for those people who go to the pub every tuesday. You could say that you implement a goToThePubEveryTuesday interface, which makes you eligible to receive $5 beers at the pub on tuesdays because your "contract" enforces you to implement a method called "RecieveHappyHourBeersOnTuesday". Mmmmmmmmmmm beer.

Furthermore, any class that implements an interface is considered to be of the type of that interface. So if you have a function that expects a parameter to be of the type goToThePubEveryTuesday, you could in actual fact pass it an instance of any class that implements the goToThePubEveryTuesday interface, as the interface forms a part of that classes inheritance tree.

Have a look at the ICampaignBehavior interface for more details on how this particular interface affects the module system. It is quite important.

Interfaces

ICampaignBehavior

The ICampaignBehavior interface enforces a CampaignBehavior contract with its implementing class.

In reality, this interface only enforces a RegisterEvents method, but this method is very important, as it is used by the module system to hook into the regular game execution loop for custom modules a.k.a mods.

When loading up modules, the module system looks for any classes that extend CampaignBehaviorBase, or implement the ICampaignBehavior interface, and it calls the RegisterEvents method on those classes. Therefore, this method is the entry point for your code into the TaleWorlds module system.
