Class FoodConsumptionBehavior

This CampaignBehaviour defines how your troops consume food units during gameplay. It is a good example of how to extend the CampaignBehaviorBase class.

As you can see it implements the RegisterEvents method as described in the ICampaignBehavior interface. This means that when the module is loaded, this classes RegisterEvents method will be called, allowing this class to hook into the module system.

the guts of this class is mainly in the OnTick event, which runs once every frame (I think). It checks if your party is starving, and if so, calls its own PartyConsumeFood method.

The PartyConsumeFood method takes a couple of things into consideration and then decides if it can slaughter some livestock or not, and then adjusts your party morale accordingly.
