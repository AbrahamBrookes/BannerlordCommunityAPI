Class MissionObject

The MissionObject is a ScriptComponentBehaviour that is attached to a GameEntity(TaleWorlds.Engine.GameEntity.html) in order to make it a placeable, interactable object in the "mission" world - that is, the first/third person world, not the world map.

The MissionObject inherits from ScriptComponentBehaviour. You would also say that MissionObject extends ScriptComponentBehaviour. This means that all of the functions and properties available in ScriptComponentBehaviour are also available in MissionObject. This is what people mean when they say "Inheritance".

MissionObject is also an abstract class. That means that you can't ever create a new MissionObject. In order to use this class, you must extend it, and your extending class must implement any abstract methods that are defined in this abstract class.

A good example of an object that extends MissionObject would be Bird.

REMEMBER! Bird is still a ScriptComponentBehaviour, because it extends a class that extends that class. So a Bird is still added to a GameEntity as any other ScriptComponentBehaviour can be, and it has access to all of the methods and properties of both MissionObject and ScriptComponentBehaviour. This is the beauty of inheritance!
